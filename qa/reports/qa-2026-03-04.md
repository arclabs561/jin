## 4. Recall Quality (Core Quality Gate)

### 4a. HNSW recall fix (STRUCTURAL fix applied)

**Root cause**: Two bugs in HNSW layer assignment:
1. `assign_layer` used geometric coin-flip loop instead of `floor(-ln(u) * mL)` per Malkov & Yashunin 2018 (Algorithm 1, line 4).
2. `m_l` default was `1/ln(2) ~= 1.44` instead of `1/ln(M) = 1/ln(16) ~= 0.36`.

Together these placed ~50% of nodes at layer 2+, destroying hierarchical structure. Fix applied in `src/hnsw/graph.rs`.

**Before**: 30-55% recall@10 (thresholds set to 0.30-0.55)
**After**: 65-100% recall@10 typical (non-deterministic; thresholds raised to 0.60-0.90)

### 4b. Test-derived recall thresholds (updated)

|------|---------|--------|-----------|---------|--------|
